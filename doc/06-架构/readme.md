# 架构设计

## 一、中文 / 国内推荐书籍

- 《前端架构：从入门到微前端》 — 黄峰达
  系统介绍前端架构设计、组件化、微前端、架构演进等。
- 《前端架构师：基础建设与架构设计思想》 — 侯策
  更偏思想 +建设性基础设施，从工程化、基础建设、技术规范等角度出发。
- 《微前端设计与实现》 — Luca Mezzalira 著
  深入讲解微前端架构的实现模式、团队协作、模块拆分、部署策略等。
- 《前端技术架构与工程》
  包含工程化、构建、组件化、性能、前后端分离等内容。

## 二、英文 / 国际优秀书籍（架构方向）

下面是一些英文书籍，非常适合思考大规模前端系统、可维护性、长期演进、团队协作等：

- Frontend Architecture for Design Systems
  专注于如何为设计系统构建可扩展、可维护的前端架构。适合负责设计系统或 UI 库团队。
- Micro Frontends Architecture for Scalable Applications
  探讨微前端架构在大型组织 / 多团队中的应用，如何拆分、如何管理依赖、如何部署。
- Micro Frontends in Action
  实战导向，案例 +代码 +策略兼具，对想落地微前端的人非常有帮助。
- Modern Front‑End Architecture
  对现代前端架构趋势进行总结，包括模块化、构建、状态管理之类。
- Front‑End Reactive Architectures
  探讨响应式 /声明式架构：例如信号（signal）、数据驱动、反应式编程在前端架构中的应用。
- The Art of Micro Frontends
  更高层次、战略视角：从组织、团队、系统设计出发看微前端。
- Frameworkless Front‑end Development
  探讨在不依赖大型框架（React/Vue 等）下，如何设计自己模块化、可维护的前端架构。
- Frontend Development: The Ultimate Guide
  较全能的一本书，覆盖前端开发方方面面（JS、构建、性能、架构等），适合架构新人参考。

## 三、理论类 +软件架构通用书（对前端架构也非常有启发）

- Pattern-Oriented Software Architecture（设计模式 +架构风格）
  虽然不是专门讲前端，但设计模式和架构思想可以很容易迁移到前端架构设计中。

## 四、小结 +建议

- 新晋架构师 /想负责架构的人：先读国内的《前端架构：从入门到微前端》，把微前端、演进式架构这些现实问题都理解一遍。
- 负责设计系统/组件库：推荐 Micah Godbolt 的书，它聚焦在设计系统＋架构。
- 做大型团队 /大组织：微前端书籍 +架构艺术书籍非常适合。
- 想思考更底层的架构思想：可以搭配设计模式 +通用软件架构书一起看。
