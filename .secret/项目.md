优秀贡献奖

【热力图】
价值创造：
 可视化展示页面元素的点击量数据，辅助产品、开发角色优化产品体验，减轻开发者的 Web 埋点工作量。
个人贡献：

 1. 编码实现。核心开发者，解决多种场景下页面绘制问题，兼容普通 DOM 节点、Web Component、自定义 DOM 等场景，归档到公司知识资产中；跨技术栈的构建产物，可以平滑地对接到各类项目中。
 2. 数据处理优化。初版热力图的数据清洗、过滤，完全由前端处理，从数据加载到页面渲染。改由后端处理后，整个流程由改造前的 7 - 10 秒，优化到 3 - 5 秒。
 3. 团队协作。针对百万级的数据清洗、过滤流程，设计解决方案，并辅助后端开发者执行方案落地。
 4. 平台对接。辅助对接到铸剑平台、星河大数据平台等多个平台中，直观展示各平台中业务功能的使用状况。

【代码编辑器 - 代码格式化】
价值创造：
 按照归纳的格式化标准，将 SQL 代码规范化进行美化，为使用在线编辑器的开发者提供开发便利，同时为代码分析提供底层支撑。
个人贡献

 1. 归纳规则。收集不同引擎下 SQL 编码风格，发现规律，归纳总结为代码格式化的规范，形成格式化标准。
 2. 编码实现。核心开发者，梳理需求，设计解决方案，改造开源代码，支持
 3. 复用代码。
 4. 团队协作。为后端开发者提供脚本，支持离线解析 SQL；为铸剑开发者提供源码，支持在铸剑平台实现统一格式化风格。

【热力图】
主动发起创新
实际业务价值：

 1. 排除低效无用需求
  超 10 个不同部门的产品，调研中表示在业务中均不同程度能减少低效需求
  案例：
  1.1 销售行为分析，日访问量 8000 以上，想给所有标签页面增加汇总和均值功能，但从热力图点击次数中发现，使用频次非常少，故取消该投入产出比较低的功能。
 2. 减少常规埋点需求；超 4 个不同部门的产品或开发，调研中确认已减少埋点开发量；
 3. 调动并提高产品、开发等各个角色参与页面优化的意识；
 4. 每单个案例，在帮助开发和产品在需求中，至少减少了 1 天的工作量；
创新的可复制性：
 可复制难度中等。
 前端处理时，需要完全还原原始页面，涉及到 Web Compoent、Canvas 、自定义 DOM 等场景，市面上未有工具能够完全处理这些场景。热力图项目，可以兼容这些场景。
 后端处理时，数据级别达到百万量级，数据的清洗、过滤涉及的业务场景较多，内部处理逻辑复杂。
是否全面应用：
 目前已对接到铸剑平台、星河大数据平台的 7 个项目中。MAU约 220 人。

代码质量

1. 热力图项目中，采用webpack、bable等手段，将包体积由500Kb减小至300Kb，减少了40%的包体积。
2. 在SQL代码规范专项中，将改造后的代码格式化源码，共享给基础服务部，在产研代码报表中复用；构建为commonjs模块，共享给后端开发者复用。

开发效率

1. 在SQL代码规范专项中，深入并改造编辑器源码，实现了代码折叠与预览、语义大纲与自定义格式化等复杂功能，并能抽离出功能提供给基础服务部，复用至产研代码报表中，提高业务开发效率。
2. 在热力图项目中，使用此工具帮助产品排除低效无用需求、帮助开发减少常规埋点需求，同时调动并提高产品、开发等各个角色参与页面优化的意识。不同程度地提高了各项目地开发效率。

价值点1：排除低效无用需求。超 10 个不同部门的产品，问卷调研中表示在业务中均不同程度能减少低效需求。举例：销售行为分析，业务想给所有标签页面增加汇总和均值功能，但从热力图点击次数中发现，使用频次非常少，最终取消该低效需求。

价值点2：减少常规埋点需求。从需要产品收集、开发、查看约1-2天工作量，到缩短或无需工作量，直接查看。案例：超 4 个不同部门的产品或开发，问卷调研中确认已减少埋点开发量。举例：crm客户管理的产品同学，要求前端给需要看点击数据的位置增加自定义埋点，后期推荐使用该热力图后取消了该需求。近期上线的反爬系统，由于接入热力图，也无需开发进行自定义埋点。

价值点3：调动并提高产品、开发等各个角色参与页面优化的意识。问卷调研或跨声反馈中，不少业务主管等同事反馈认可该数据，表示该热力图可以辅助他们跟进功能使用情况，进行功能下线或改配策略。

价值点4：提高前端开发创新意识。该项目已分别获得 1 个GUI外观专利奖，1 个发明专利奖。
